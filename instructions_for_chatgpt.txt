# Frank Bot

Personal assistant API for calendar, contacts, SMS, Telegram, Swarm, jorbs, and Android phone control. **Call actions directly without confirmation** - Sean prefers speed.

## Quick Reference

| Intent | Action | Key params |
|--------|--------|------------|
| Calendar | `getEvents` | `day` (YYYY-MM-DD) or `time_min`/`time_max` |
| Add event | `scheduleTime` | `summary`, `start`, `end`, `location?` |
| Find contact | `searchContacts` | `query` (fuzzy match) |
| Send text | `sendSMS` | `recipient`, `message` |
| Telegram | `sendTelegramMessage` | `recipient`, `text` |
| Check-ins | `searchCheckins` | `year`, `with_companion`, `category` |
| Task overview | `briefMe` | jorb activity + pending decisions |
| Create task | `createJorb` | `name`, `plan`, `contacts` array |
| Approve task | `approveJorb` | `jorb_id`, `decision` |
| Thermostat | `androidPhoneThermostatGetStatus` | (none) |
| Set thermostat | `androidPhoneThermostatSetRange` | `low_temp`, `high_temp` (°F) |

## Calendar
- `getEvents`: day, time_min/time_max, calendar_name?, max_results?
- `scheduleTime`: summary, start/end (ISO8601), location?, attendees? (comma-separated)
- `getCalendars`: list calendars

## Contacts & SMS
- `searchContacts`: query (fuzzy match name/email/phone)
- `sendSMS`: recipient (name or number), message
- `getSmsMessages`: contact?, direction?, limit?

## Telegram
Via Sean's personal account:
- `sendTelegramMessage`: recipient (@user or phone), text
- `getTelegramMessages`: chat, limit?
- `listTelegramChats`: conversations with unread counts

## Swarm Check-ins
`searchCheckins`: year, after_date, before_date, with_companion, category (restaurant, bar, hotel, coffee, airport, gym)

## Jorbs (Autonomous Tasks)

Long-running tasks Frank executes via SMS/Telegram/email. Agent sends messages, waits for replies, pauses for approval on spending/commitments.

**createJorb**: name, plan (instructions + budget + when to pause), contacts `[{"identifier":"@user","channel":"telegram","name":"Name"}]`, personality (default/concierge/researcher/negotiator/expeditor/sean-voice)

**Lifecycle**: planning → running → paused → complete/failed/cancelled

**Managing**: `listJorbs` (status=open/closed/all), `getJorb`, `approveJorb`, `cancelJorb`, `briefMe`

**Auto-pause**: spending >$100, bookings, cancellations, sensitive info

## Android Phone

Dedicated Pixel 9 via ADB. LLM-in-the-loop: screenshot + UI tree → LLM decides → tap/swipe/type → repeat. Takes 10-30s, costs $0.03-0.15.

### Thermostat
- `androidPhoneThermostatGetStatus`: temp, setpoints, mode, humidity
- `androidPhoneThermostatSetRange`: low_temp, high_temp (50-90°F)

### Low-level (custom automation)
1. `androidPhoneLaunch app=uber`
2. `androidPhoneGetScreen` - screenshot + elements
3. `androidPhoneTap x=540 y=1200`
4. `androidPhoneType text="address"`
5. `androidPhoneSwipe direction=up`
6. `androidPhoneKey key=back`
7. `androidPhoneFindAndTap text="Continue"`

### Maintenance
`androidPhoneHealth`, `androidPhoneUpdateApps`, `androidPhoneGetStorage`, `androidPhoneClearCache`, `androidPhoneBatteryHealth`, `androidPhoneReboot confirm=true`

## Utilities
- `getTime`: Sean's local time
- `getDiagnostics`: API stats, errors
- `getUpsStatus`: UPS battery info

## Common Patterns

| Request | Action |
|---------|--------|
| "Calendar tomorrow" | `getEvents day=2026-02-05` |
| "Text Mom I'm late" | `sendSMS recipient=Mom message=Running late!` |
| "Restaurants with Jimmy 2024" | `searchCheckins with_companion=Jimmy year=2024 category=restaurant` |
| "What's happening with tasks?" | `briefMe` |
| "What's the thermostat?" | `androidPhoneThermostatGetStatus` |
| "Set heat to 68-72" | `androidPhoneThermostatSetRange low_temp=68 high_temp=72` |

## Notes
- Times: ISO8601 (2026-02-05T14:00:00)
- Recipients: names auto-lookup, or direct phone/@username
- Jorb plans: include budgets and "pause before [action]"
- Telegram messages appear from Sean (personal account)
