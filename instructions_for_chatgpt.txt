## API Structure

This API uses a "learn-then-execute" pattern. For each capability:
1. Call the `*ApiLearn` endpoint FIRST to understand what's available
2. Then use the task/action endpoints

## Quick Reference

| Family | Learn First | Then Use |

## Scripts (Access Calendar, SMS, Swarm, Telegram, etc.)

**frankScriptApiLearn** returns comprehensive docs about:
- `frank.calendar` - events, create events, list calendars
- `frank.contacts` - search by name/email/phone
- `frank.sms` - send SMS messages
- `frank.swarm` - Foursquare check-in history
- `frank.telegram` - send/receive Telegram messages
- `frank.ups` - UPS battery status
- `frank.time` - current time with timezone

**Workflow:**
1. `frankScriptApiLearn` - learn what frank.* can do
2. `frankScriptTaskStart` with code + params
3. `frankScriptTaskStatus` to poll for result

**Example - Get tomorrow's calendar:**
```json
{
  "slug": "get-calendar",
  "code": "def main(frank, day):\n    return frank.calendar.events(day=day)",
  "params": {"day": "2026-02-05"}
}
```

**Example - Send SMS:**
```json
{
  "slug": "send-text",
  "code": "def main(frank, to, msg):\n    return frank.sms.send(to, msg)",
  "params": {"to": "Mom", "msg": "Running late!"}
}
```

## Android Phone

Dedicated Pixel 9 via LLM-in-the-loop automation.

**androidPhoneApiLearn** returns docs about:
- Thermostat control (Nest)
- Smart home (lights, etc.)
- Ride services (Uber, Lyft)
- Food delivery (DoorDash)
- Device maintenance
- Browser/general apps

**Workflow:**
1. `androidPhoneApiLearn` - learn capabilities
2. `androidPhoneTaskDo` with goal → returns task_id
3. `androidPhoneTaskGet` to poll for result

**Examples:** "Check thermostat", "Set lights to 50%", "What's Uber price to SFO?"

**Safety:** STOPS before payments/purchases. Cost: $0.03-0.15, takes 10-30s.

## Jorbs (Autonomous Tasks)

Long-running tasks Frank executes via SMS/Telegram/email.

**jorbsApiLearn** returns docs about:
- Lifecycle (planning → running → paused → complete)
- Contacts format: `[{identifier, channel, name}]`
- Personalities: default, concierge, researcher, negotiator, expeditor
- Auto-pause rules

**Workflow:**
1. `jorbsApiLearn` - learn how jorbs work
2. `jorbCreate` with name, plan, contacts
3. `jorbsBrief` for overview + pending decisions
4. `jorbApprove` to approve/reject paused actions

**Example plan:** "Book dinner for 4 at a nice Italian restaurant in Dallas for Saturday 7pm. Budget $50-80/person. Pause before confirming."

## Claudia (Coding Assistant)

AI assistant for discussing and implementing code changes.

**claudiaApiLearn** returns docs about:
- Available repositories
- Chat workflow (create → discuss → end → execute)
- Prompt execution

**Workflow:**
1. `claudiaApiLearn` - see repos, learn workflow
2. `claudiaChatCreate` with repo_name, title
3. `claudiaChatMessage` to discuss
4. `claudiaPromptExecute` to implement
