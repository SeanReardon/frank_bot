# Claudia Progress Log

This file tracks completed work by the Claudia autonomous agent.
Each entry documents what was implemented and lessons learned.

## Codebase Patterns

- FrankAPI is in meta/api.py - patch target for tests is `meta.api.FrankAPI`
- Scripts for jorbs are Python expressions (eval first, exec fallback), not main() functions
- JorbStorage.add_script_result() stores results as JSON in jorbs table's script_results column
- Agent loop in process_jorb_event() re-creates JorbSession each iteration for fresh context
- Rate limiting uses `iter_{jorb_id}` key in _message_counts dict, tracked per-jorb
- SCRIPT_EXECUTION_TIMEOUT=300s, MAX_ITERATIONS_PER_HOUR=20, MAX_ITERATIONS_PER_DAY=100
- Pre-existing test failures: test_enrich_event_sms_with_contact, test_send_sms_message (missing google module)
- Integration tests use real JorbStorage with temp DB, mock only LLM and FrankAPI
- Action priority in JorbSession._parse_response: done > pause > script > no_action
- FrankAPI namespace pattern: class with methods using lazy imports + _run_async(), property accessor on FrankAPI
- actions/__init__.py eagerly imports all action modules - patch() for actions not in __init__.py needs explicit `import actions.foo` first
- conftest.py mocks missing modules (telethon, telnyx, ntplib) so patch targets resolve
- TelegramBotListener uses httpx GET /bot{token}/getUpdates with offset tracking + allowlist filtering
- Channel type is `Literal["telegram", "telegram_bot", "sms", "email"]` in jorb_storage.py and message_buffer.py
- conftest.py has autouse fixture `disable_switchboard_mode` — tests needing switchboard must re-enable with monkeypatch
- MessageBuffer supports metadata dict on BufferedMessage/BufferedEvent (for telegram_bot_chat_id)
- _should_autocreate_jorb treats telegram_bot identically to telegram
- api_learn_action test bypasses actions/__init__.py via importlib.util.spec_from_file_location
- Vault client: VAULT_TIMEOUT_SECONDS env var (default 30s) caps entire retry sequence
- app.py degraded mode: settings_degraded flag + fallback Settings when Vault unreachable
- Docker healthcheck start_period=60s accommodates Vault retry delays
- entrypoint.sh: ADB_TIMEOUT (10s) and VAULT_TIMEOUT (15s) env vars for bounding commands
- Screenshot persistence: _persist_screenshot() writes to ./data/screenshots/{task_id}.png with 0o600 perms
- TelegramClientService.send_photo() validates paths via _PHOTO_ALLOWED_PREFIXES (./data/screenshots/, /tmp/android_screenshot_*)
- _execute_task_background lazy imports: patch at source module (services.android_task_storage, services.android_phone_runner), not at actions.android_phone
- Sean's Telegram username is @SeanReardon (from services/telegram_allowlist.py)
